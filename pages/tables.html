Tabeller

Below are examples of tables used in the interface. One is suitable for long lists, and the other for tables with many columns.

Long List Table

This table style is meant for displaying a long list of items clearly.

# Name Category
1 Item One Category A
2 Item Two Category B
3 Item Three Category A 

<table class="table-long">
    <thead>
        <tr><th>#</th><th>Name</th><th>Category</th></tr>
    </thead>
    <tbody>
        <tr><td>1</td><td>Item One</td><td>Category A</td></tr>
        <tr><td>2</td><td>Item Two</td><td>Category B</td></tr>
    </tbody>
</table>

<thead> stands for 'table header' i.e. it groups the uppermost row of the table that contains header cells.

<tr> stands for 'table row'.

<th> stands for 'table head'; but unlike <thead>, represents a single cell.

<td> stands for 'table data' and contains the informational content of a cell.

<tbody> groups the body content in a table.

Wide Table (Many Columns)

This table style is suitable for displaying data with multiple columns.

ID Name Category Status Date Notes
001 Item One A Active 2025-11-19 First entry
002 Item Two B Inactive 2025-11-18 Second entry 

<table class="table-wide">
    <thead>
        <tr><th>ID</th><th>Name</th><th>Category</th><th>Status</th><th>Date</th><th>Notes</th></tr>
    </thead>
    <tbody>
        <tr><td>001</td><td>Item One</td><td>A</td><td>Active</td><td>2025-11-19</td><td>First entry</td></tr>
        <tr><td>002</td><td>Item Two</td><td>B</td><td>Inactive</td><td>2025-11-18</td><td>Second entry</td></tr>
    </tbody>
</table>
